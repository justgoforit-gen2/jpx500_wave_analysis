# Requirements: Phase 6 — 資産バリュータブ（有価証券・土地の含み損益）

## 背景

jpx500_wave_analysis の Streamlit UI に「資産バリュー」タブを追加し、
EDINET 有報から有価証券・土地の含み損益を自動算出して表示する。

対象銘柄: 日産自動車（7201 / E02142）を検証銘柄として実装。

## 要件

### 機能要件

1. **Stage 2: B/S サマリー取得**
   - EDINET API から最新有価証券報告書を取得
   - 投資有価証券・土地・現金の B/S 簿価を表示

2. **Stage 3: 政策保有株式リスト取得**
   - iXBRL から特定投資株式・みなし保有株式を取得
   - 銘柄名・株数・簿価・時価（有報記載）を表形式で表示

3. **Stage 4: 設備の状況（土地）取得**
   - 有報 HTML テーブルから事業所別土地面積・簿価・所在地を取得
   - 親会社分（7拠点）を対象とする

4. **Stage 5: 有価証券の時価換算**
   - yfinance で現在株価を取得
   - 含み損益（時価 − 簿価）を算出

5. **Stage 7: 土地の時価換算（倍率方式）**
   - 簿価 × multiplier（スライダーで調整）で時価推計
   - 含み損益を算出

6. **資産バリューサマリー**
   - 時価総額・現金・有価証券時価・土地時価・含み益合計を集計表示

### 非機能要件

- EDINET API 呼び出しは `@st.cache_data(ttl=3600)` でキャッシュ
- XBRL パースは `@st.cache_data(ttl=86400)` でキャッシュ
- ZIP は `naibu-ryuho-app/scripts/raw/` に保存（スクリプトと共有）
- `EdinetClient` は sys.path injection で再利用（コード重複なし）

### スコープ外（Phase 2 以降）

- Stage 6: 国交省 不動産情報ライブラリ API（REINFOLIB_API_KEY 要取得）
- 複数銘柄の一括処理・ランキング化
- 有利子負債を使ったネットキャッシュ計算
- 過去5年の時系列推移グラフ
