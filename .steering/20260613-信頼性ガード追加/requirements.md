# 要求内容: 資産バリュー分析 — 信頼性判別ガード追加

## 概要

`asset_value_tab.py` に銀行・保険会社を自動判別するガードを追加し、製造業専用パーサーを金融機関に誤適用して信頼性ゼロのレポートが出力されるリスクを排除する。

## 背景

資産バリュー分析タブを銀行（京都フィナンシャルグループ E38714）で試したところ、以下の2つの問題が発生した：

1. **有価証券**: 銀行は「投資有価証券」科目を持たず、Stage 3/5 の分析が機能しない
2. **土地（設備の状況）**: 銀行の設備テーブルは列構造が製造業と異なるため、パーサーが誤った列を読み込み、土地簿価合計が B/S 値の 1 万倍になった

「おかしな値と気づかずに使う」リスクを排除するため、判別式とガードを追加する。

## 実装対象の機能

### 1. EDINET 府令コード取得
- `search_doc_id()` の return dict に `ordinance_code` フィールドを追加
- EDINET API レスポンスの `ordinanceCode` を取得（`"010"` = 一般、`"030"` = 銀行、`"040"` = 保険）

### 2. 会社種別分類ヘルパー `_classify_filer()`
- 府令コード → 名前キーワードの優先順位で `"bank"` / `"insurance"` / `"general"` を返す
- 銀行キーワード: `銀行`, `フィナンシャルグループ`, `信用金庫`, `信用組合`
- 保険キーワード: `保険`, `生命`, `損保`, `共済`

### 3. 土地サニティチェック `_land_sanity()`
- `Σ 事業所土地簿価 > B/S 土地簿価 × 5` の場合、信頼できないと判定
- ordinanceCode が `"010"` でも設備テーブルが壊れている会社への後出しガード

### 4. UI ガード処理
- `skip_securities`: 銀行・保険の場合は Stage 3/5 をスキップして警告表示
- `use_bs_land_only`: 銀行・保険またはサニティ NG の場合は B/S 簿価 × 倍率のみ表示
- `_render_bs_land_fallback()`: 共通フォールバック表示ヘルパー関数

## 受け入れ条件

### 銀行検出
- [ ] 京都フィナンシャルグループ（E38714）を入力すると `⚠️ 銀行・金融グループ` 警告が表示される
- [ ] Stage 3/5（有価証券）が「対象外」メッセージでスキップされる
- [ ] Stage 4/7（土地）が B/S 簿価 × 倍率表示に切り替わる
- [ ] 従来の 449 兆円のような異常値が出なくなる

### リグレッション
- [ ] 日産（E02142）で従来通り動作する（スキップされない）

### サニティチェック
- [ ] `ratio > 5` の場合にエラーメッセージが表示される
- [ ] サニティ NG 時も B/S 倍率方式にフォールバックして処理継続する

## 成功指標

- 銀行・保険会社に対して誤解を招く数値が表示されなくなる
- 製造業への影響なし（リグレッションなし）

## スコープ外

- 銀行専用の有価証券分析（時価評価済み資産の分析）
- 保険会社専用の資産分析
- 複数銘柄の一括判別
- 府令コード `"030"` / `"040"` 以外の業種コードへの対応
