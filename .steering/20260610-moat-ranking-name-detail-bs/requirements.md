# 要求内容

## 概要

Moat Score「ランキング」タブと「銘柄詳細」ビューの実用性を高める 3 つの UI 強化を行う。具体的には (1) ランキング表への銘柄名列追加、(2) 行クリックでの銘柄詳細へのジャンプ、(3) 銘柄詳細への貸借対照表 (BS) 表示の追加。

## 背景

- 現在のランキングタブ (`_tab_moat_ranking`, `app.py`) は 4 桁の `code` しか表示せず、銘柄名がないため一覧の可読性が低い。
- 行クリックでの遷移がなく、案内文は「コードをコピーして Moat Score タブに貼り付けてください」という手作業を強いている。`docs/functional-design.md` F-04 は「行クリック → Moat Score タブへ遷移」と記載するが未実装。
- 銘柄詳細ビュー (`show_detail_view`) には業績 (P/L) はあるが BS がなく、財務健全性（自己資本比率・有利子負債など）を確認できない。naibu-ryuho-app の `financial_metrics` テーブルには BS 全項目が既に存在し、`modules/naibu_client.py` が読み取り専用で直読みしている。

## 実装対象の機能

### 1. ランキング銘柄名列
- ランキング表（全体表 + 今朝の Top10）に「銘柄名」列を `code` の直後に表示する。
- `load_stock_list()` の `code/name/ticker` を moat_scores に left マージして取得する。
- ユーザーはコードを知らなくても企業名で銘柄を識別できる。

### 2. 銘柄詳細へのジャンプ
- ランキング全体表の行をクリックすると、その銘柄の詳細ビュー (`show_detail_view`) へ遷移する。
- 既存の `_navigate_to_detail(code, name, ticker)` + `st.session_state["view"]="detail"` パターンを流用する。
- 遷移先は **Moat Score タブではなく銘柄詳細ビュー**（技術的理由: `st.tabs` はプログラム的にタブ切替不可、`view="detail"` オーバーレイは session_state で遷移可能）。
- ユーザーはランキングから 1 クリックで個別銘柄のチャート・業績・BS を確認できる。

### 3. 銘柄詳細の BS 表示
- 銘柄詳細ビューの業績 (P/L) ブロック直後に「貸借対照表 (BS・最新期)」セクションを追加する。
- 表示は最新本決算期のスナップショット: (a) 指標カード（総資産/純資産/総負債/自己資本比率/現金/有利子負債）、(b) 構成グラフ（資産 vs 負債+純資産の積み上げ横棒）、(c) 明細表。
- データは `naibu_client.fetch_balance_sheet(code)` で `financial_metrics` から読み取り専用直読み。
- ユーザーは個別銘柄の財務健全性を視覚的に把握できる。

## 受け入れ条件

### 銘柄名列
- [ ] ランキング全体表に「銘柄名」列が表示され、トヨタ等の正しい企業名が出る
- [ ] 今朝の Top10 表にも銘柄名列が表示される
- [ ] stock list に存在しない code でも表示が壊れない（NaN/空欄で継続）

### 銘柄詳細へのジャンプ
- [ ] ランキング全体表の行クリックで銘柄詳細ビューに遷移する
- [ ] 遷移後、`code name` タイトルと「← 一覧に戻る」ボタンが表示される
- [ ] ticker 欠損銘柄ではクラッシュせず警告表示で安全にガードされる

### BS 表示
- [ ] `fetch_balance_sheet("7203")` が total_assets/total_equity/total_liabilities 等を返す
- [ ] BS データのない銘柄では `None` を返し、UI は「BS データなし」キャプションを表示
- [ ] 銘柄詳細に総資産/純資産/自己資本比率の指標カード・構成グラフ・明細表が描画される
- [ ] 自己資本比率 = total_equity / total_assets が正しく計算・表示される

## 成功指標

- ランキング → 銘柄詳細 → BS 確認が 1 クリック導線で完結する
- BS 取得は単一行クエリで `show_detail_view` の再描画を体感的に遅くしない（`@st.cache_data` ラップ）

## スコープ外

以下はこのフェーズでは実装しません:

- 複数年の BS 推移グラフ（最新期スナップショットのみ）
- naibu API への新規エンドポイント追加（DB 直読みで完結）
- recompute / 書込系処理の追加（GET/読込のみ）
- Top10 表からの行クリック遷移（全体表のみで十分）
- `data/`・`*.db`・`*.parquet` の移動・書換え
