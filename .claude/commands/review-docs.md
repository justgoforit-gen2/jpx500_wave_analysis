# ドキュメントレビュー

引数: ドキュメントパス（例: `/review-docs HANDOVER.md`）

## 手順

### ステップ1: ドキュメントの存在確認

指定されたドキュメントが存在するか確認する。

### ステップ2: doc-reviewer サブエージェント起動

Task ツールで `doc-reviewer` サブエージェントを起動:
- `subagent_type`: "doc-reviewer"
- `description`: "Document detailed review"
- `prompt`: "[ドキュメントパス] を詳細にレビューしてください。\n\n以下の観点で評価してください:\n1. 完全性: 必要な項目が全て含まれているか\n2. 具体性: 曖昧な表現がないか\n3. 一貫性: 他のドキュメント（HANDOVER.md、code、API 実装）と整合しているか\n4. 実装可能性: 開発者が実装に必要な情報が揃っているか\n\nレビューレポートを作成してください。"

### ステップ3: レビュー結果の要約

サブエージェントが作成したレビューレポートの要点を抽出し、ユーザーに報告する。

## 出力形式

```markdown
# ドキュメントレビュー結果

## ドキュメント: [ファイル名]

### 主な改善点

1. [改善点1] (優先度: 高/中/低)
2. [改善点2] (優先度: 高/中/低)
3. [改善点3] (優先度: 高/中/低)

### 総合評価

[1-5]/5

### 次のアクション

- [推奨対応1]
- [推奨対応2]

詳細なレポートはサブエージェントの出力を参照してください。
```

## 注意事項

- レビューは詳細な分析のため、数分かかる場合があります
- サブエージェントは独立したコンテキストで動作するため、メインエージェントのコンテキストは消費しません
